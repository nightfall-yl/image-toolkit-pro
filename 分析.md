# ImgBox Pro 功能分析文档

## 一、设置页结构概览

插件设置分为 4 个主要导航选项卡：
1. **通用** - 插件显示与自动处理
2. **图片本地化** - 附件下载、压缩、命名和保存路径
3. **图片清理** - 未使用附件清理
4. **图片预览** - 右键菜单、拖拽缩放、点击预览

---

## 四、功能重复/重叠分析

### 4.1 重复设置项

#### 1. **删除去向设置（3处重复）**

| 位置 | 设置项 | 配置字段 |
|------|--------|----------|
| 图片清理 | 已删除文件的去向 | `clearUnusedDeleteOption` |
| 图片预览 | 附件删除去向 | `attachFlowDeleteOption` |
| 图片清理 | 彻底删除文件 | `removeOrphansCompl` |

**问题分析**：
- `clearUnusedDeleteOption` 和 `attachFlowDeleteOption` 功能完全一致（永久删除/Obsidian回收站/系统回收站）
- `removeOrphansCompl` 是布尔值，与上述两个设置逻辑重叠
- **建议**：合并为一个统一的"删除去向"设置

#### 2. **压缩设置（2处重复）**

| 位置 | 设置项 | 配置字段 | 适用场景 |
|------|--------|----------|----------|
| 图片本地化 | 压缩图片（网络图片） | `PngToJpeg` | 下载的网络图片 |
| 图片本地化 | 压缩图片（粘贴图片） | `PngToJpegLocal` | 粘贴/拖入的图片 |

**问题分析**：
- 两个设置功能几乎相同，只是应用场景不同
- 压缩格式 (`ImgCompressionType`) 和质量 (`JpegQuality`) 是共享的
- **建议**：合并为"启用图片压缩"，通过来源区分行为，或统一处理所有图片

#### 3. **日志弹窗设置（2处重复）**

| 位置 | 设置项 | 配置字段 |
|------|--------|----------|
| 图片清理 | 显示清理日志弹窗 | `clearUnusedLogsModal` |
| 图片预览 | 显示删除日志弹窗 | `attachFlowLogsModal` |

**问题分析**：
- 两者都是控制是否显示操作后的日志弹窗
- **建议**：合并为统一的"显示操作日志弹窗"设置

#### 4. **调试模式设置（2处重复）**

| 位置 | 设置项 | 配置字段 |
|------|--------|----------|
| 通用-开发者选项 | 核心调试模式 | `VERBOSE` |
| 图片预览 | 预览调试模式 | `attachFlowDebug` |

**问题分析**：
- 都是控制控制台日志输出
- **建议**：合并为统一的"调试模式"设置，或按模块细分但集中管理

### 4.2 功能重叠模块

#### 1. **图片清理 vs 图片预览(AttachFlow) 的删除功能**

| 模块 | 功能 | 命令/入口 |
|------|------|-----------|
| 图片清理 | 清理未使用图片/附件 | Ribbon 按钮、命令面板 |
| 图片预览 | 删除当前笔记及其附件 | 右键菜单、命令面板 |

**问题分析**：
- 两者都涉及"删除附件"，但场景不同
- 图片清理是批量扫描删除，AttachFlow 是针对当前笔记的快捷删除
- **建议**：保留两者，但统一删除去向设置

#### 2. **自动处理设置分散**

| 位置 | 设置项 | 功能 |
|------|--------|------|
| 通用 | 自动处理 | 总开关 |
| 通用 | 自动处理间隔 | 时间间隔 |
| 通用 | 处理所有新建 Markdown 文件 | Markdown 处理开关 |
| 通用 | 处理所有新附件 | 附件处理开关 |

**问题分析**：
- 4个设置相互关联但分散在通用选项卡中
- **建议**：合并为一个"自动处理"分组，包含启用开关、间隔、处理范围

### 4.3 配置字段命名不一致

| 功能 | 字段名 | 问题 |
|------|--------|------|
| 自动处理间隔 | `realTimeUpdateInterval` | 命名不清晰，应为 `autoProcessInterval` |
| 自动处理 | `realTimeUpdate` | 应为 `autoProcess` |
| 处理新建 Markdown | `processCreated` | 应为 `processNewMarkdown` |
| 处理新附件 | `processAll` | 命名不清晰，应为 `processNewAttachments` |

### 5.1 设置项合并建议

| `clearUnusedDeleteOption` + `attachFlowDeleteOption` + `removeOrphansCompl` | `deleteDestination` | 统一删除去向 |

### 5.3 建议删除/简化的功能

1. **"不创建 Obsidian 附件文件夹"** (`DoNotCreateObsFolder`)
   - 用途：兼容其他插件
   - 建议：作为高级选项隐藏或移除，默认行为应兼容

2. **"添加原始文件名或打开文件标签"** (`addNameOfFile`)
   - 用途：在替换后的标签后追加原始文件名
   - 建议：评估使用频率，低频功能可移除

3. **"日期格式"** (`DateFormat`)
   - 用途：媒体文件夹模板中的日期格式
   - 建议：如果 `${date}` 模板使用率低，可移除该设置

4. **部分附加命令**
   - "Set the first found # header as a note name"
   - "Convert selection to URI"
   - "Convert selection from html to markdown"
   - 建议：这些与核心功能无关，可作为独立插件或移除
