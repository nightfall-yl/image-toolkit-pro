# ImgBox Pro 功能分析文档

## 一、文档说明

本文档基于当前代码实现整理，重点分析设置页中的 4 个主模块：

1. **通用**
2. **图片本地化**
3. **图片预览**
4. **图片清理**

分析范围以当前实现为准，主要参考：

- `src/config.ts`
- `src/settingstab.ts`
- `src/main.ts`
- `src/contentProcessor.ts`
- `src/clearUnusedUtils.ts`
- `src/attachflowFeature.ts`
- `src/attachflowDeleteAll.ts`

---

## 二、整体结构概览

当前版本的设置页已经整理为 4 个主导航模块：

1. **通用**
   负责插件总开关类行为、自动处理流程、界面入口和开发者选项。
2. **图片本地化**
   负责外链图片、本地文件、base64 图片和新附件的本地化、压缩、命名、路径与存储策略。
3. **图片预览**
   负责图片右键菜单、单击预览、拖拽缩放、图片回跳原笔记，以及“删除当前笔记及其附件”这类 AttachFlow 交互能力。
4. **图片清理**
   负责扫描并清理未使用图片、未使用附件，以及当前笔记目录中的未关联附件。

与旧版本相比，当前结构的特点是：

- 删除去向已统一为 `deleteDestination`
- 日志弹窗已统一为 `showOperationLogs`
- 调试模式已统一为 `debugMode`
- 图片压缩已统一为 `compressImage`
- 开发者选项不再单列为主模块，而是折叠在“通用”里

## 三、4 个模块功能分析

### 3.1 通用

`通用` 模块主要负责插件的全局行为控制，以及自动处理相关流程。

#### 主要设置项

| 设置项 | 配置字段 | 当前作用 |
|--------|----------|----------|
| 显示通知 | `showNotifications` | 控制处理完成后是否显示通知 |
| 显示图片清理 Ribbon 图标 | `showCleanupRibbon` | 控制左侧 Ribbon 是否显示“清理未使用图片”入口 |
| 自动处理 | `autoProcess` | 控制是否启用自动处理 |
| 自动处理间隔 | `autoProcessInterval` | 控制自动处理队列的轮询间隔 |
| 处理所有新建 Markdown 文件 | `processNewMarkdown` | 新建或同步得到的 Markdown/Canvas 类文件是否自动处理 |
| 显示批量命令 | `showBatchCommands` | 控制批量本地化和当前目录清理命令是否显示 |

#### 折叠分组

`通用` 模块当前包含两个折叠区域：

1. **批量命令**
   用于控制批量命令是否显示。

2. **开发者选项**
   用于控制调试输出。

开发者选项中的设置如下：

| 设置项 | 配置字段 | 当前作用 |
|--------|----------|----------|
| 调试模式 | `debugMode` | 控制插件整体调试输出 |

#### 实际行为

`通用` 模块除了提供设置项，还直接影响以下行为：

- 控制是否注册和显示清理 Ribbon 图标
- 控制是否显示批量命令
- 控制是否开启自动处理队列
- 控制新建文件、修改文件、粘贴内容时是否自动触发本地化流程
- 控制新附件是否自动移动到插件管理的目标目录

#### 模块定位

一句话概括：`通用` 是插件的“运行策略中心”，决定插件何时工作、怎样自动工作，以及向用户暴露哪些入口。

### 3.2 图片本地化

`图片本地化` 是插件的核心模块，负责将图片和附件转为可管理的本地资源，并改写笔记中的引用。

#### 主要设置项

| 设置项 | 配置字段 | 当前作用 |
|--------|----------|----------|
| 单个附件重试次数 | `downloadRetryCount` | 下载失败时的重试次数 |
| 处理所有新附件 | `processNewAttachments` | 新附件是否进入本地化与迁移流程 |
| 下载未知文件类型 | `downloadUnknownTypes` | 是否允许下载未知文件类型 |
| 压缩图片 | `compressImage` | 是否压缩下载或新建得到的图片 |
| 压缩格式 | `compressionFormat` | 压缩输出格式，支持 `JPEG` / `WebP` |
| 图片质量 | `compressionQuality` | 压缩质量 |
| 文件大小下限（KB） | `minFileSizeKB` | 小于阈值的文件不下载 |
| 排除扩展名 | `excludedExtensions` | 这些扩展名的附件不参与下载 |
| 新附件使用时间+MD5命名 | `useTimestampNaming` | 新粘贴或拖入的附件是否使用 `YYYYMMDD-HHmmss-md5前6位` 命名 |
| 新附件保存位置 | `attachmentSaveLocation` | 跟随 Obsidian / 保存到指定根目录 / 保存到笔记旁边 |
| 媒体文件夹 | `mediaFolderPath` | 媒体资源保存目录模板 |
| 标签中的路径写法 | `linkPathFormat` | 支持完整路径、相对路径、仅文件名 |
| 日期格式 | `dateFormat` | `${date}` 模板的格式 |
| 保留链接标题 | `preserveCaptions` | 改写链接时保留标题/说明 |

#### 高级选项

`图片本地化` 模块中，以下低频能力已下沉到折叠的“高级选项”区域：

| 设置项 | 配置字段 | 当前作用 |
|--------|----------|----------|
| 添加原始文件名或打开文件标签 | `appendOriginalName` | 处理完成后追加原始文件名或打开文件标签 |
| 同步移动、删除或重命名媒体文件夹 | `syncMediaFolder` | 在特定模式下同步媒体目录变化 |
| 不创建 Obsidian 附件文件夹 | `skipObsidianFolderCreation` | 用于兼容其他插件工作流 |

#### 当前支持的处理对象

当前实现支持处理以下来源：

1. 网络图片链接（`http` / `https`）
2. 本地文件链接（`file://`）
3. base64 图片
4. 新建或拖入的附件
5. 粘贴进编辑器后触发的图片处理流程

#### 当前核心能力

`图片本地化` 模块当前包含这些核心能力：

1. **下载与保存**
   将外链图片或本地文件读取为二进制数据，并保存到目标媒体目录。

2. **压缩与转码**
   对符合条件的图片执行压缩，当前主要围绕 PNG 输入，输出为 JPEG 或 WebP。

3. **去重与重命名**
   基于内容哈希检测重复文件；开启时间命名时，可生成 `YYYYMMDD-HHmmss-md5前6位` 风格的新文件名。

4. **路径改写**
   根据当前设置，将原始链接改写为 Obsidian 附件链接或 Markdown 链接，并支持不同路径策略。

5. **新附件迁移**
   对 Obsidian 默认目录中新创建的附件进行重命名、压缩、迁移，并同步更新当前笔记中的引用。

6. **媒体目录同步**
   在“保存在笔记旁边的指定文件夹”模式下，笔记被删除或重命名时，可同步处理对应媒体目录。

#### 相关命令

| 命令名称 | 作用 |
|----------|------|
| `Localize attachments for the current note (plugin folder)` | 本地化当前笔记中的附件，使用插件配置目录策略 |
| `Localize attachments for the current note (Obsidian folder)` | 本地化当前笔记中的附件，跟随 Obsidian 默认附件目录 |
| `Localize attachments for all your notes (plugin folder)` | 批量处理全部笔记中的附件 |

#### 模块定位

一句话概括：`图片本地化` 是插件的“资源落地与链接改写引擎”。

### 3.3 图片预览

`图片预览` 模块承接了 AttachFlow 相关交互能力，是当前插件的“附件交互增强层”。

#### 主要设置项

| 设置项 | 配置字段 | 当前作用 |
|--------|----------|----------|
| 单击预览图片 | `clickPreviewEnabled` | 点击图片中间区域打开预览 |
| 自适应显示比例 | `previewAdaptiveRatio` | 控制大图预览时的自适应缩放比例 |
| 拖拽缩放图片 | `dragResizeEnabled` | 在源码模式或实时预览模式下启用拖拽缩放 |
| 缩放步进 | `dragResizeStep` | 控制拖拽缩放的最小刻度 |

#### 当前核心能力

1. **图片右键菜单增强**
   为图片或附件提供扩展菜单项。

2. **单击看大图**
   点击图片中间区域可弹出预览层，并支持再次点击关闭。

3. **自适应缩放**
   大图预览时可根据设置比例自动缩放到合适尺寸。

4. **滚轮缩放与拖拽浏览**
   预览中的图片支持滚轮缩放和拖动。

5. **拖拽调整图片尺寸**
   在源码模式或实时预览模式下，可直接拖拽改变图片显示尺寸。

6. **跳转到原笔记**
   在图片导航栏或图片标签页的右键菜单中，可跳转回引用该图片的原笔记。

7. **删除当前笔记及其附件**
   提供 `Delete Current Note and Its Attachments` 命令；删除时会跳过仍被其他笔记引用的附件，避免误删共享资源。

#### 相关命令

| 命令名称 | 作用 |
|----------|------|
| `Delete Current Note and Its Attachments` | 删除当前笔记及其仅被当前笔记引用的附件 |

#### 模块定位

一句话概括：`图片预览` 是插件的“图片交互、附件操作与来源导航模块”。

### 3.4 图片清理

`图片清理` 模块负责识别并删除不再被引用的图片或附件，同时也负责清理当前笔记目录中的未关联附件。

#### 主要设置项

| 设置项 | 配置字段 | 当前作用 |
|--------|----------|----------|
| 删除去向 | `deleteDestination` | 统一控制删除到永久删除、Obsidian 回收站或系统回收站 |
| 显示操作日志弹窗 | `showOperationLogs` | 操作完成后是否显示日志弹窗 |
| 排除文件夹 | `excludedFolders` / `excludedFoldersRegexp` | 这些目录中的文件不会被自动处理，清理时也会跳过 |
| 清理时排除子文件夹 | `excludeSubfolders` | 清理时是否连同排除目录下的子目录一起忽略 |

#### 当前核心能力

1. **清理 Vault 中未使用图片**
   扫描全库图片文件，找出未被引用的图片并删除。

2. **清理 Vault 中未使用附件**
   扫描全库所有附件文件，找出未被引用的附件并删除。

3. **清理当前笔记目录中未关联附件**
   在“保存在笔记旁边的指定文件夹”模式下，检测当前笔记附件目录中那些不再被该笔记引用的文件，并执行清理。

4. **统一删除策略**
   当前所有清理相关删除都依赖统一字段 `deleteDestination`，不再分别维护多套删除配置。

5. **统一日志展示**
   清理完成后可统一显示日志窗口，而不是按不同模块分散控制。

#### 当前引用识别方式

当前实现并不只是简单看 Markdown 正文中的标准链接，而是综合使用多种来源判断附件是否被引用：

- `metadataCache.resolvedLinks`
- Markdown 文件中的 embeds / links
- frontmatter 中的 banner 或图片路径
- Markdown / Canvas 文本中的图片链接
- Canvas 节点中的文件引用

这意味着当前的“未使用判断”比旧版更完整，特别是对 frontmatter 和 canvas 内容的支持已经纳入清理逻辑。

#### 相关命令

| 命令名称 | 作用 |
|----------|------|
| `Clear Unused Images in Vault` | 清理全库中未使用的图片 |
| `Clear Unused Attachments in Vault` | 清理全库中未使用的附件 |
| `Clear Unlinked Attachments in Current Note Folder (Next to Note mode)` | 清理当前笔记附件目录中未关联的附件 |

#### 模块定位

一句话概括：`图片清理` 是插件的“全库引用检查与删除执行模块”。

## 四、模块边界总结

从当前实现看，4 个模块的边界已经比旧版本清晰很多：

| 模块 | 主要职责 | 典型入口 |
|------|----------|----------|
| 通用 | 总体行为控制、自动处理、入口显示、开发者选项 | 设置页、自动事件、Ribbon 控制 |
| 图片本地化 | 下载、压缩、去重、命名、保存、链接改写 | 命令面板、自动处理、粘贴处理 |
| 图片预览 | 右键交互、预览、缩放、附件移动、跳转原笔记 | 图片右键菜单、点击、命令面板 |
| 图片清理 | 引用扫描、未使用附件识别、删除执行 | Ribbon、命令面板、日志弹窗 |

当前模块划分的优点：

1. `通用` 不再混杂业务细节，更偏向全局行为控制
2. `图片本地化` 集中承载“资源落地”相关逻辑
3. `图片清理` 集中承载“引用判断和删除”逻辑
4. `图片预览` 集中承载 AttachFlow 交互增强能力

## 七、结论

当前版本的 `ImgBox Pro` 已经形成了较清晰的 4 模块结构：

1. `通用` 负责全局运行策略
2. `图片本地化` 负责资源下载、落地与链接改写
3. `图片预览` 负责 AttachFlow 式交互增强
4. `图片清理` 负责引用扫描和删除执行

从代码状态看，这次整理已经完成了旧文档中提出的多项合并目标，尤其是删除策略、日志弹窗、调试模式和图片压缩开关的统一。当前这份结构可以作为后续继续优化设置页、编写 README、规划重构边界的基准文档。
